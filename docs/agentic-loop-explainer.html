Agentic Loop Fix

Why Gemini returned an empty response — and how the loop fixes it

What changed

Before — broken

1
User sends English message "I go to school yesterday."
↓
2
Gemini decides to call a function Calls save_english_mistake — uses its entire response turn for the function call, generates no text
↓
3
Backend saves mistake to Firestore Stream ends. fullText = ""
↓
4
User sees empty bubble An empty assistant message is saved and displayed

After — fixed

1
User sends English message "I go to school yesterday."
↓
2
Gemini calls the function (no text yet) Provider intercepts it, saves save_english_mistake to queue, yields StreamItem to route
↓
3
Provider sends function result back New step. Because hasText = false, provider calls sendMessageStream([{ functionResponse: ... }]) — tells Gemini the function succeeded
↓
4
Gemini generates its text response Streamed and yielded normally
↓
5
User sees a real reply Mistake saved silently; conversation continues naturally

Result

All 35 tests pass

When Gemini calls save_english_mistake with text in the same turn → no second call needed, works as before.
When Gemini calls it with no text (the bug) → provider loops, Gemini replies, user sees a normal response.
